📡 Why Agent Requests Aren't Showing Up

Your app is hosted at aizen993.github.io/Qmobility-Command-Hub/ — that's great! The issue is Supabase Row Level Security (RLS).

The Problem: Supabase requires RLS policies to allow agents to read/write data without logging in. If RLS blocks anonymous reads, agent submissions fail silently.
Fix in 2 minutes:
1. Open app.supabase.com → Your project → Authentication → Policies
2. For the weeks table → Add policy → "Enable read access for all users" (anon role)
3. For the weeks table → Add policy → "Enable update for all users" (anon role)
4. For the workspaces table → Add policy → "Enable read access for all users" (anon role)
5. Save & test — agents should now sync instantly.

🌐 Host Online — Agent Links

Your agent links use file:/// which only works on your laptop. To share with agents, you need to host this file online. Free options:

Option 1: Netlify Drop (easiest — 30 seconds)
1. Go to app.netlify.com/drop
2. Drag and drop your YousefShtawiSchedule.html file
3. You'll get a URL like https://xyz-abc.netlify.app
4. Agent links become: https://xyz-abc.netlify.app/YousefShtawiSchedule.html?agent=ID
Option 2: GitHub Pages (your current setup)
Your repo: aizen993.github.io/Qmobility-Command-Hub/

To activate agent links:
1. Go to your repo → Settings tab (top menu)
2. Scroll to Pages section (left sidebar)
3. Under "Source", select Deploy from a branch
4. Select branch: main, folder: / (root)
5. Click Save
6. Wait 1-2 minutes → your site is live!
7. IMPORTANT: Your HTML file must be named index.html
→ Rename YousefShtawiSchedule.html to index.html in the repo
8. Agent links become: https://aizen993.github.io/Qmobility-Command-Hub/?agent=ID
Option 3: Tiiny.host (instant)
1. Go to tiiny.host
2. Upload the HTML file → Get a shareable link instantly

Once hosted, all agent links will work from any device. The app still uses Supabase for data, so everything syncs automatically.

⚡ Genesys Cloud Integration

Connect to Genesys Cloud via a Cloudflare Worker proxy — free, no server needed, live data every 30s.

Quick Setup (10 min, completely free):
1. In Genesys: Admin → Integrations → OAuth → Add Client → Client Credentials → add "Read-only Administrator" role
2. Go to workers.cloudflare.com → free account → Create Worker
3. Paste genesys-worker.js into the editor → Save & Deploy
4. Worker Settings → Variables → add: GENESYS_CLIENT_ID, GENESYS_CLIENT_SECRET, GENESYS_REGION=mec1.pure.cloud
5. Paste the Worker URL above (no trailing slash) → click Test
📄 Full step-by-step guide included in download: genesys-setup-guide.html
